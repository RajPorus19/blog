**Codecrafters** challenge: rebuild Redis from scratch in Go to master network protocols, data persistence, and replication.

### Challenges completed
- ✅ **RESP** protocol parsing (Redis Serialization Protocol)
- ✅ Basic commands (GET, SET, DEL, etc.)
- ✅ RDB persistence (Redis Database file format)
- ✅ Master-slave replication
- ✅ Concurrency management with goroutines

### Tech stack
- **Go** — main language
- **TCP Sockets** — network communication
- **RESP Protocol** — binary parsing
- **Goroutines & Channels** — concurrency

### Key learnings
- Redis architecture: how a cache is designed deep down
- Managing concurrent connections in Go
- Complexities of replication in distributed systems
